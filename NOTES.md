# 项目重要注意事项

## 项目结构
1. proto 文件生成的 pb.go 文件已经存在于 `common/proto/gen` 目录下
2. 不要重新生成 proto 文件，除非确实需要修改 proto 定义
3. 项目结构已经更改，所有服务都使用 `common` 目录下的共享代码

## Windows 环境注意事项
1. Windows PowerShell 不支持 `&&` 命令连接符
2. 需要分步执行命令
3. 使用 `;` 作为命令分隔符（如果支持）

## 开发注意事项
1. 当前阶段不需要客户端代码，使用其他工具进行测试
2. 所有服务都应该使用 `common` 目录下的共享代码
3. 修改 proto 文件后需要重新生成 pb.go 文件时，确保生成到正确的目录

## 常见错误
1. 不要在错误的目录下生成 proto 文件
2. 不要重复创建已经存在的客户端代码
3. 不要使用 Windows 不支持的命令语法
4. 确保使用正确的导入路径
